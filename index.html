Selected work

Client_Work_Event_Analysis_Pipeline

A reporting pipeline for recurring CSV datasets where formats and event categories change over time, but the reporting output needs to remain comparable.

  • Input: raw CSV files with inconsistent structure
  • Process: parsing, cleaning, validation, and normalization
  • Output: consistent summary tables ready for Excel/Sheets reporting

InfraWatchLite

A lightweight Linux health monitor that samples CPU, memory, and disk usage and writes timestamped logs with warnings when thresholds are crossed.

  • Input: system resource metrics from the host machine
  • Process: periodic sampling, threshold checks, and logging
  • Output: plain-text logs and warnings that show system health over time

JournalDM

A private journaling web app built end-to-end so entries move from a simple UI into an API and database with predictable storage.

  • Input: user text entries submitted through the web interface
  • Process: API handling, validation, and database writes
  • Output: a persistent journal history that can be queried and displayed

HelixNet

A machine-learning pipeline that classifies protein secondary structure (helix/sheet/coil) with reproducible training and evaluation outputs.

  • Input: labeled protein sequences
  • Process: encoding, model training, and evaluation
  • Output: trained model artifacts and metrics (F1 ≈ 0.89)
